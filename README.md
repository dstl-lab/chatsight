# Chatsight

A research tool for analyzing student-AI chatlogs. Instructors can browse conversations from an AI tutoring system, generate semantic labels using Google Gemini, and iteratively refine labeling with steering feedback.

---

## Tech Stack

| Layer | Tech |
|-------|------|
| Frontend | React 19 + TypeScript, Vite 6, Tailwind CSS v4, Recharts, React Markdown |
| Backend | FastAPI, Uvicorn, SQLModel |
| Local DB | SQLite (`chatsight.db`) |
| External DB | PostgreSQL (read-only, student chatlogs) |
| AI | Google Gemini 2.0 Flash (function calling) |

---

## Prerequisites

- Node.js (for frontend)
- Python 3.11+ with [uv](https://github.com/astral-sh/uv) (for backend)
- `kubectl port-forward` to `localhost:5432` — the external PostgreSQL database is accessed through this tunnel

## Architecture

### Data Flow

```
External PostgreSQL (dsc10_tutor_logs)
        │  read-only
        ▼
Backend assembles chatlog content (Student / Assistant messages)
        │
        ▼
Frontend displays transcript in TranscriptPanel
        │
        ▼  user clicks "Generate Labels"
Backend calls Gemini 2.0 Flash via function calling
        │
        ▼
Gemini returns structured labels (message_index, label, evidence, rationale, granularity)
        │
        ▼
Labels saved to local SQLite → rendered in LabelsPanel + BehaviorTimeline
        │
        ▼  user provides steering notes + clicks "Re-process"
New LabelSet row created (history preserved) → repeat from Gemini call
```

### Two Database Connections

**Local SQLite** (`chatsight.db`) — auto-created at startup:
- `LabelSet` — one row per labeling run (including re-runs with steering notes)
- `Label` — individual labels within a set

**External PostgreSQL** (`dsc10_tutor_logs`) — read-only:
- Single `events` table with columns: `id`, `event_type`, `user_email`, `payload` (JSONB), `created_at`
- Event types: `tutor_query` (student message) and `tutor_response` (AI reply)
- Conversations grouped by `payload->>'conversation_id'`
- Stable chatlog ID = `MIN(event.id)` per conversation
- Notebook filename read from `payload->>'notebook'` on response events

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/chatlogs` | All conversations (no cap), grouped by conversation_id |
| `GET` | `/api/chatlogs/{id}` | Single chatlog with its latest label set |
| `GET` | `/api/chatlogs/{id}/label-sets` | Full label history for a chatlog |
| `POST` | `/api/label` | Generate labels (body: `chatlog_id`, `steering_notes`) |

## AI Labeling

`label_service.py` calls **Gemini 2.0 Flash** with a forced function call (`mode=ANY`) so output is always structured JSON. The prompt asks Gemini to:

1. Infer context (subject area, assignment, student level)
2. Label each student–AI interaction turn with:
   - **label** — short semantic tag (e.g. "Concept Probe", "Misconception", "Clarification Request")
   - **evidence** — a brief quote from the transcript
   - **rationale** — why this label applies
   - **granularity** — `high` (broad behavior) / `mid` (specific skill) / `low` (surface detail)

If steering notes are provided, they are injected as a `## Steering Instructions` block in the user message.

Each "Re-process with Feedback" call creates a **new** `LabelSet` row, preserving the full history of labeling runs.

## Frontend Details

- **State** lives entirely in `App.tsx` — no global state library.
- **Vite** proxies `/api` to `http://localhost:8000` in dev, so no CORS configuration is needed.
- **Tailwind v4** is configured via the `@tailwindcss/vite` plugin only — no `tailwind.config.ts`.
- **Sync highlighting**: `highlightedIndex` is shared between `TranscriptPanel`, `LabelsPanel`, and `BehaviorTimeline`. Clicking a label scrolls the transcript; clicking a message shows its labels; clicking a bar in the timeline does both.
- **Sidebar grouping**: chatlogs can be grouped by notebook (default, collapsed) or by student, with a search box that filters across both fields.
- **Markdown rendering**: AI responses in the transcript are rendered as markdown using `react-markdown` + `@tailwindcss/typography`.
